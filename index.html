System Architecture
Three steps to
total knowledge.
A deterministic pipeline that transforms a single name into a comprehensive, linked intelligence profile — recursively, idempotently, relentlessly.
01
Seed
Ingest
A minimal CSV — first name, last name, class year. That's all the system needs. One row per target. The pipeline consumes the queue and begins resolution.
02
LLM Policy 1
Family Discovery
Autonomous identification of immediate family members. Creates or merges family_files — shared JSON records keyed by deterministic family_id. Iteratively expands outward with each pass.
03
LLM Policy 2
Profile Enrichment
Deep enrichment of every individual in the family tree. Assets, education, career, affiliations, internet accounts — structured, linked, and persistent. Per-member or whole-family execution modes.
Intelligence Policies
Two autonomous policies.
Zero data loss.
Each policy is idempotent, append-only, and context-aware. The more they know, the more they find.
LLM_Policy_1 → p1
Family Tree Constructor
Extracts immediate family members from any input — a bare name or an existing family_file. Produces candidate members with confidence scores. Returns higher recall when personal_data from P2 is available. Strictly append-only: never overwrites, never corrupts.
Idempotent Append-Only Recursive Context-Aware
LLM_Policy_2 → p2
Profile Enrichment Engine
Consumes a family_file and outputs structured personal_data deltas. Operates per-member for efficiency or whole-family for maximum context exploitation. Shared-asset linking propagates updates bidirectionally across households.

Future Architecture
Graph_DF

The end state: a complete graph theory representation of every digital footprint in the dataset. Every person is a node. Every relationship is an edge. Every data point is an attribute. The family_file and relationship_edges are the foundation — Graph_DF is the weapon.

Built on stable, idempotent policies.
Scales to any population size.
Every new data point makes the whole graph smarter.

Core Properties
Engineered for
relentless reliability.
Idempotent
Run any policy any number of times. Same input, same output. No corruption, no duplication, no side effects.
Append-Only
Data only grows. Policies never overwrite, delete, or mutate existing records. The family_file is an immutable ledger.
Context-Amplified
Each policy improves with more data. P1's recall increases when P2's personal_data is present. The system gets smarter with every cycle.
Public Data Only
Every data point sourced exclusively from publicly available records. Social media, professional footprints, open registries.
Bidirectional Linking
Shared-asset linking propagates updates across households. Relationship edges maintain distinct inverse-aware directionality.
Domain Portable
Architecture is domain-agnostic. Entity resolution and relationship mapping applicable to any population, any dataset, any context.
